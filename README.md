# sistema-pos-facturacion-backend
Sistema backend del proyecto rematePOS para gestión de inventario, ventas y facturación.
- Arquitectura: Microservicios 
- Metodología: Scrum 
- Backend: Java + Spring Boot
- Frontend: React
- DB: MongoDB
- Integración DIAN: Proveedor tecnológico autorizado vía API REST

Los comercios minoristas dedicados a la cacharrería y a los remates manejan una gran variedad de productos y un alto volumen de ventas diarias, lo que dificulta el control eficiente del inventario, el registro de ventas y la trazabilidad de las operaciones. En muchos casos, estos procesos se realizan de forma manual o mediante herramientas aisladas, generando errores, inconsistencias en la información y dificultades para la toma de decisiones.

Adicionalmente, la implementación de la facturación electrónica representa un reto para este tipo de negocios, debido a la complejidad técnica y normativa del proceso y a la necesidad de integrarse con proveedores tecnológicos autorizados por la DIAN. Esto evidencia la necesidad de un sistema POS que integre ventas, inventarios y facturación electrónica bajo una arquitectura moderna, escalable y preparada para su uso comercial.
